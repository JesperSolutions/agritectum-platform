# Feature Descriptions

This directory contains completed feature documentation and archived specifications.

## Directory Structure

```
10-feature-descriptions/
├── completed/              # Completed feature documentation
├── archived-specifications/ # Archived specifications from NEW_FEATURES_SPECIFICATION.md
└── README.md              # This file
```

## Purpose

This directory serves as the repository for:

1. **Completed Features** - Documentation of features that have been implemented
2. **Archived Specifications** - Original specifications moved from NEW_FEATURES_SPECIFICATION.md

## Workflow

### When a Feature is Completed:

1. **Extract Feature Documentation**
   - Copy feature section from `NEW_FEATURES_SPECIFICATION.md`
   - Create new file: `completed/FEATURE_NAME.md`

2. **Remove from NEW_FEATURES_SPECIFICATION.md**
   - Delete the feature section
   - Update the table of contents
   - Update feature count

3. **Archive Original Specification**
   - Move original specification to `archived-specifications/`
   - Name: `FEATURE_NAME_SPECIFICATION_YYYY-MM-DD.md`

4. **Update Documentation**
   - Update `NEW_FEATURES_SPECIFICATION.md` version
   - Update this README
   - Update main docs README

### File Naming Convention

**Completed Features:**

- `OFFER_AND_ACCEPTANCE_FLOW.md`
- `PRICING_AND_VARIABLE_CALCULATIONS.md`
- `AUTOMATIC_REMINDERS_AND_WEATHER_LOGIC.md`

**Archived Specifications:**

- `OFFER_AND_ACCEPTANCE_FLOW_SPECIFICATION_2025-01-15.md`
- `PRICING_AND_VARIABLE_CALCULATIONS_SPECIFICATION_2025-01-20.md`
